1. 좋은 프롬프트란?
- 모델이 의도한 방식으로 일관성 있게 응답할 수 있도록 도와주는 지침이다.
- 명확한 목적, 입력 형식, 출력 기대 형식을 담고 있어야 한다.

2. 명확하고 구체적으로 작성하라
- 막연한 요청보다는 세부적인 지시를 제공해야 모델의 정확도가 높아진다.
- (X) 이 문장을 고쳐줘 → (O) 아래 문장의 맞춤법, 띄어쓰기, 문장 부호 오류를 교정해줘

3. 출력 포맷을 명시하라
- 원하는 출력이 문장인지, JSON인지, 단락인지 구체적으로 지시하라.
- 예) 반드시 수정된 문장만을 출력하세요.

4. 모델의 역할을 정의하라
- system 메시지를 통해 모델의 역할이나 말투, 스타일을 사전 정의할 수 있다.
- 예) "너는 한국어 문법 전문가야. 정확하고 간결하게 답변해."

5. 불필요한 정보는 제거하라
- 예시가 너무 많거나, 부연 설명이 길면 모델이 혼동할 수 있다.
- 프롬프트는 가능한 한 짧고 집중적으로 유지하라.

6. 멀티턴 전략
- 복잡한 작업은 단계별로 나눠 처리하라 (ex: 띄어쓰기 → 맞춤법 → 문장 부호)
- 각 턴마다 명확한 요청을 설정하고 컨텍스트를 유지하라.

7. 예시 활용
- 적절한 예시는 학습된 출력 패턴을 유도하는 데 효과적이다.
- 예) 문장: 나는밥을먹엇다 → 나는 밥을 먹었다

8. 금지된 출력 지시
- 다음 문구는 사용하지 말 것: “직접 고쳐보세요”, “정답은 다음과 같습니다”, “설명드리겠습니다”

9. 토큰 수 제한 고려
- 전체 프롬프트가 2000 토큰을 초과하지 않도록 주의하라
- 반복된 지시나 중복 문장 제거

10. 실험 전략
- 프롬프트를 유형별로 나눠 실험 (간결형 / 나열형 / 전문가 스타일 / 공손형 등)
- 리워드 기준을 토큰 효율성 + 정확도로 설정하면 개선 가능성 높음
